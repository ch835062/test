## ⚙️ n8n 與 Docker 環境佈署
* **n8n 的本質**
    * 它不是單純的軟體，而是一個 **Server（伺服器）**，應保持 24 小時啟動以執行自動化任務。
    * 名稱由來：名稱中包含 8 個 'n'，故簡稱為 `n8n`。
    * 特性：**No-code / Low-code**，透過「節點」連線取代編寫程式碼。
* **Docker 容器技術**
    * **Container（容器）**：像是一台「虛擬的小電腦」，不佔用本機過多資源，且可隨時開啟或關閉。
    * **Image（映像檔）**：像安裝光碟，用來建置伺服器的模板。
    * **Volume（資料卷）**：獨立儲存專案資料的空間。即便刪除容器，只要保留 Volume，工作流就不會遺失。
* **環境啟動指令**
    * 啟動 n8n 伺服器指令（需開啟終端機執行）：
    * ### Mac / Linux 版本 (多行格式)
      ```bash
         docker run -it --rm \
          --name n8n \
          -p 5678:5678 \
          -v n8n_data:/home/node/.n8n \
          -e N8N_ENFORCE_SETTINGS_FILE_PERMISSIONS=true \
          -e GENERIC_TIMEZONE="Asia/Taipei" \
          docker.n8n.io/n8nio/n8n
      ```
     * ### Windows 版本 (單行合併格式)
      ```
      docker run -it --rm --name n8n -p 5678:5678 -v n8n_data:/home/node/.n8n -e N8N_ENFORCE_SETTINGS_FILE_PERMISSIONS=true -e GENERIC_TIMEZONE="Asia/Taipei" docker.n8n.io/n8nio/n8n
      ```
    * 存取網址：`http://localhost:5678`。
    * 注意事項：第一次進入需設定 Email 與密碼（**密碼務必記住，無法找回**），並至左下角 Setting 進行「激活 (Activation)」以啟用 Personal 功能。

## 🧠 JSON 資料格式核心（重要觀念）
* **資料傳遞語言**
    * 節點之間傳遞資料的統一格式為 **JSON**。
    * 就像是在不同生產線步驟間傳遞的一張「數位表格」。
* **JSON 語法元素**
    * **Array（陣列）**：使用 `**[]**` 包裹，代表多筆資料。編號（Index）從 `0` 開始。
    * **Object（物件）**：使用 `**{}**` 包裹，代表單筆資料的細節（如：一個人的聯絡資訊）。
    * **Key & Value（鍵與值）**：成對出現，例如 `"Name": "Nathan"`，Key 是欄位名稱，Value 是實際內容。
* **資料類型標示**
    * **String（字串）**：文字內容，必須加上雙引號 `""`。
    * **Number（數值）**：數字，不加引號。
    * **Boolean**：真假值 `**true**` 或 `**false**`。
    * **null**：代表該欄位內容為**空值**。

## 🪜 n8n 實作操作流程
* **工作流 (Workflow) 基礎**
    * 一個專案稱為一個「**工作流**」，內含多個功能方塊（**節點**）。
    * 註解工具：使用 **Sticky Note（便利貼）** 撰寫流程說明，支援 Markdown 語法（如 `#` 代表大標題）。
* **常用節點操作**
    * **Trigger（觸發器）**：流程的起點，例如 `*Manually Trigger*`（手動點擊執行）。
    * **Data Store**：範例資料源，可使用 `*Customer Data Store*` 抓取測試用的五筆人員資料。
    * **Edit Fields（過濾與編輯）**：
        * 用於重新命名欄位或篩選需要的資訊。
        * **Expression（運算式）**：使用 `{{ $json.欄位名 }}` 動態抓取前一站的資料 。
* **執行與預覽**
    * 點擊 `*Execute Step*` 或 `*Execute Workflow*` 來測試節點是否正確運作。
    * 輸出預覽有三種模式：**Table（表格）**、**Schema（架構）**、**JSON（原始程式碼）**。
