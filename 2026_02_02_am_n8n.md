# n8n 自動化流程與地端 AI 入門

## 📘 課程主題總覽
* **核心目標**：進入 **No-code** 自動化世界，串接不同軟體服務（如 **Google Drive**, **Gmail**, **AI**）。
* **學習成就**：
    * 成功架設地端 **n8n 伺服器**。
    * 掌握 **JSON** 資料傳遞格式。
    * 建立首個自動化 **工作流 (Workflow)**（抓取與過濾資料）。

## 🧠 核心觀念
* **n8n (No-code / Low-code)**：自動化管家，將手動操作轉為自動流程。
* **伺服器 (Server)**：24 小時啟動提供服務的運算單元。
* **Docker (容器技術)**：獨立的虛擬小電腦，不影響主系統設定。
* **節點 (Node)**：流程中的單一步驟（觸發、抓取、發送）。
* **JSON**：數位化表格格式，由「鍵 (Key)」與「值 (Value)」組成。
* **觸發器 (Trigger)**：流程起點（時間、訊息、手動）。

## ⚙️ 環境準備
* **作業系統**：Windows / Mac 均可。
* **必要工具**：
    * **Docker Desktop**：運行伺服器的核心環境。
        * [官方下載網址](https://www.docker.com/products/docker-desktop/)
    * **瀏覽器**：建議使用 **Chrome**（需關閉自動翻譯功能）。
    * **Ollama**：地端 **AI** 實作工具。
* **安裝確認**：看到綠色 **Status** 或小鯨魚圖示代表啟動成功。

## 🪜 操作步驟
1.  **Step 1：啟動 n8n 伺服器容器**
    於終端機執行 **Docker** 指令。
2.  **Step 2：開啟 n8n 網頁控制台**
    連線網址：`http://localhost:5678`
3.  **Step 3：建立新的工作流 (Workflow)**
    點擊「Create workflow」並命名。
4.  **Step 4：新增觸發節點 (Trigger)**
    搜尋並選擇「Manually Trigger」。
5.  **Step 5：抓取範例資料**
    使用「Customer Data Store」節點，執行「Get All People」。
6.  **Step 6：過濾與轉換資料欄位**
    使用「Edit Fields」節點進行欄位重新命名。

## 💻 指令 / 程式碼區

### Mac / Linux 版本 (多行格式)
```bash
docker run -it --rm \
 --name n8n \
 -p 5678:5678 \
 -v n8n_data:/home/node/.n8n \
 -e N8N_ENFORCE_SETTINGS_FILE_PERMISSIONS=true \
 -e GENERIC_TIMEZONE="Asia/Taipei" \
 docker.n8n.io/n8nio/n8n

## 💻 指令 / 程式碼區 (續)

### Windows 版本 (單行合併格式)
```powershell
docker run -it --rm --name n8n -p 5678:5678 -v n8n_data:/home/node/.n8n -e N8N_ENFORCE_SETTINGS_FILE_PERMISSIONS=true -e GENERIC_TIMEZONE="Asia/Taipei" docker.n8n.io/n8nio/n8n

### 💻 指令關鍵參數說明
* **`-p 5678:5678`**： **門牌對接**，讓網頁能連進容器。
* **`-v n8n_data...`**： **資料持久化**，刪除容器後專案仍會保留。
* **`-e GENERIC_TIMEZONE`**： 設定時區為**台北 (Asia/Taipei)**。

---

### ⚠️ 常見錯誤與排除
* **密碼管理**： 忘記密碼需刪除 **Volume** 重新設定，務必記牢。
* **連線失效**： 若網址打不開，請檢查 **Docker** 容器是否為 **Running** 狀態。
* **翻譯干擾**： 嚴禁開啟瀏覽器**自動翻譯**，避免程式邏輯（如 **Auto**）被誤翻。

---

### 📌 重點整理
* **資料流向**： 每個節點皆有 **Input** 與 **Output**。
* **JSON 結構**： `[]` 為**陣列 (Array)**（多筆），`{}` 為**物件 (Object)**（單筆）。
* **註解習慣**： 使用 **Sticky Note (便利貼)** 紀錄邏輯，方便日後維護。

---

### 🧩 補充與進階
* **Docker 優勢**： 易於**遷移**與**版本更新**。
* **內網穿透**： 若要讓外部服務連回地端 **n8n**，需使用 **ngrok** 或 **Cloudflare Tunnel**。
* **運算式 (Expression)**： 可使用 `{{ $json.id }}` 進行**動態資料**運算。
